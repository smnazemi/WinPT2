IPC FrameWork 1.1 is for developing applications which access Packet Tracer (PT) features from outside.This framework provides API's for remotely accessing and controlling the Packet Tracer through the IPC feature.The communication between the framework and the PT is through TCP/IP sockets. This is built around the Qt framework.

Getting Started

An application programmer can use the IPC FrameWork to get the PT components like Network, AppWindow, MultiUserManager, commandLog etc and utilize the functionalities further.A typical program using IPC Framework follows the below steps:

1. Create an instance of IPC FrameWork.
2. Get a instance of Network from IPCFrameWork().

      #include <IPCHeaders>  
      IPCFrameWork *ipcFw=new IPCFrameWork(); // Refer constructor for details.
      ipcFw->setUserInfo("net.netacad.cisco.ipctest","cisco"); 
      std::string address="localhost";   
      int port =39000;   
      try 
      {   if(m_ipcFw->startFrameWork(address,port)!=0) 
            { 
                   showMsg("Framework Not started!!!!"); 
                    return; 
            } 
            cout<<"Framework started!!!!";   
            Network *netw =ipcFw->network();   // Instance of Network   
            Device dev=netw->getDevice("Router0");// Get the Device with name  “Router0” from the network
            std::string myStr =dev.getName();   
            cout<<"\n Device Name: "<< myStr; 
      } 
      catch( IpcFwException e) 
      {   
         cout<<e.getErrMsg(); 
      }    

Notes  

1. The function CallResult IPCFrameWork::getLastCallResult() can be used to get the status of the last call.

2. The macro ENABLE_DBG_MSG_FW_ENGINE in IPCFramework.h is used to enable or disable displaying the framework engine messages on console.

3. Functions getVersion(), getBuildInfo() in IPCFramework.h are used to get the version and build information respectively.

4. Some objects of the framework are just representation of the remote object. They do not contain all the information about the remote object. It keeps an IPC call string inside. getIpcCallStr() function is used to get the call string.  When a function is invoked it uses the call strings to  send ipc call.

 To understand this, please see the example below:


  Device routerDev=ipcFw->network()->getDevice("Router0");  
  Port firstPort=routerDev.getPortAt(0);
  cout<<"\nCall Str for Port: "<< firstPort.getIpcCallStr();
  cout<<"\nPort Name: "<<firstPort.getName();
  Output:-
  Call Str for Port: network,0,getDevice,9,Router0,0,getPortAt,4,0,0,
  Port Name: Vlan1

This means that the validity of an object depends on the validity of the IPC call inside, which is NEVER updated after the creation of the object.
The object is invalid if the IPC call fails.

5. Suppose the name of the device routerDev has changed in PT(either directly from PT or through an IPC call), then the object routerDev becomes invalid.  Please see example below:

  routerDev.setName(“MYROUTER”);
  QString routerName=routerDev.getName();

  Output:-
  IPC call failed: Network:getDevice returns NULL 

getName() call will fail as the routerDev tries to get router by name "Router0" (As the ipc call string has name "Router 0") which does not exist in remote PT.


Events Mechanism

Some of the classes provide events. User gets the events through the virtual functions implemented in the extended class of the given Events Class. Use setEventCallBack() function for registering, before subscribing to events. Subscription can be done using subscribeEvent() function. You can also unsubscribe a particular event With unsubscribeEvent() function.

  class MyDeviceEvents:DeviceEvents
  { 
      public : 
      void nameChanged(QString newName, QString oldName) 
      { 
      } 
      void powerChanged(bool on) 
      { 
            if(on) 
                  cout<<"\n Device Switched on"; 
            else 
                  cout<<"\n Device Switched off"; 
      }   
      void portAdded(std::string portName) 
      { 
           cout<<"\n\n added port is:"<<"\t"<<portName.c_str(); 
      }  
      void portRemoved(std::string portName) 
      {  
            cout<<"\n\n Removed port is:"<<"\t"<<portName.c_str(); 
      } 
  };

Now, use the frame work to obtain the network object with network() function. Then pick a device (Router0 in this case) by using getDevice() function
as shown below.

  Network *netw= ipcFw->network();  
  device=netw->getDevice("Router0");

Create an instance of the class derived by you (MyDeviceEvents class in this case) Then register the class events using setEventCallBack() function. Later you   can subscribe all the events defined in the class each at a time as shown  below using subscribeEvent() function. 

            try 
            {

            MyDeviceEvents *ac=new MyDeviceEvents();   
            device.setEventCallBack((DeviceEvents*)ac);   
            device.subscribeEvent("nameChanged");   
            device.subscribeEvent("powerChanged");   
            device.subscribeEvent("portAdded");  
            device.subscribeEvent("portRemoved");   
       }   
       catch(IpcFwException e) 
       { 
           showMsg("\n"+QString::fromStdString(e.getErrMsg()));  
       }

Example Project:

An Example Project has been provided with this framework to demonstrate functionalities. Please make sure that Remote PT is listening at port number 39000 or fill the correct port number in the example application.Before testing the application drag a router device to the remote Packet Tracer work space. This will by default be named 'Router0' .Once the framework starts, you can subscribe and unsubscribe events and observe the corresponding results in the text box provided in GUI.

Steps need to be followed:

  1. Click on the “Start Framework” button to start the framework.
  2. Click on “Test FW” button to test the sample functionality of the framework (which displays port name of Router0 in this case).
  3. Click on “Subscribe” button to subscribe for the events of your choice.(powerChanged is the event Subscribed in the example).
  4. Click on “Unsubscribe” button to unsubscribe the event that was already subscribed.
